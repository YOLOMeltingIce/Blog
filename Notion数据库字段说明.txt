=====================================================
Notion 数据库字段配置说明
适用于新版 AI 博客前端界面
=====================================================

本文档说明了在 Notion 中应该创建哪些数据库以及每个数据库需要的字段。

=====================================================
一、论文读后感数据库 (Papers Database)
=====================================================

数据库名称：论文读后感 / Papers

必需字段：
-----------
1. 标题 (Title)
   - 类型: 标题 (Title)
   - 说明: 论文的标题

2. 作者 (Authors)
   - 类型: 多选 (Multi-select) 或 富文本 (Rich Text)
   - 说明: 论文作者，多个作者用逗号分隔

3. 期刊 (Journal)
   - 类型: 文本 (Text) 或 富文本 (Rich Text)
   - 说明: 发表期刊或会议名称，例如 "NeurIPS 2017"

4. 阅读日期 (Date)
   - 类型: 日期 (Date)
   - 说明: 阅读论文的日期

5. 评分 (Rating)
   - 类型: 数字 (Number) 或 选择 (Select)
   - 说明: 对论文的评分，1-5分
   - 如果使用 Select，选项为: 1, 2, 3, 4, 5

推荐字段（新版界面）：
---------------------
6. 概要 (Summary)
   - 类型: 富文本 (Rich Text)
   - 说明: 论文的简要概述

7. 阅读理由 (Reason)
   - 类型: 富文本 (Rich Text)
   - 说明: 为什么阅读这篇论文

8. 论文核心内容 (Core Content)
   - 类型: 富文本 (Rich Text)
   - 说明: 论文的核心内容和贡献

9. 我的感悟 (Insights)
   - 类型: 富文本 (Rich Text)
   - 说明: 个人对论文的理解和感悟

可选字段（兼容旧版）：
---------------------
10. 读后感 (Review)
    - 类型: 富文本 (Rich Text)
    - 说明: 整体读后感（如果没有使用新字段）

11. 关键洞见 (Key Insights)
    - 类型: 多选 (Multi-select) 或 富文本 (Rich Text)
    - 说明: 从论文中获得的关键洞见（多个用逗号分隔）

字段名称别名（脚本会自动匹配以下任一名称）：
------------------------------------------
- 标题: Title, 论文标题
- 作者: Authors, 分类
- 期刊: Journal, 会议
- 日期: Date, 阅读日期
- 评分: Rating
- 概要: Summary
- 阅读理由: Reason
- 论文核心内容: 核心内容, Core Content
- 我的感悟: 感悟, Insights
- 读后感: 内容总结, Review


=====================================================
二、项目经历数据库 (Projects Database)
=====================================================

数据库名称：项目经历 / Projects

必需字段：
-----------
1. 项目名称 (Title)
   - 类型: 标题 (Title)
   - 说明: 项目的名称

2. 项目描述 (Description)
   - 类型: 富文本 (Rich Text)
   - 说明: 项目的详细描述

3. 状态 (Status)
   - 类型: 选择 (Select)
   - 说明: 项目状态
   - 选项: 
     * completed (已完成)
     * in-progress (进行中)
     * planned (计划中)

4. 技术栈 (Technologies)
   - 类型: 多选 (Multi-select) 或 富文本 (Rich Text)
   - 说明: 使用的技术栈，例如 React, Node.js, Python 等
   - 如果用富文本，多个技术用逗号分隔

推荐字段：
-----------
5. 项目亮点 (Highlights)
   - 类型: 多选 (Multi-select) 或 富文本 (Rich Text)
   - 说明: 项目的亮点特性，多个用逗号分隔

6. GitHub (GitHub URL)
   - 类型: URL
   - 说明: GitHub 仓库链接

7. Demo (Demo URL)
   - 类型: URL
   - 说明: 项目演示链接

8. 开始日期 (Start Date)
   - 类型: 日期 (Date)
   - 说明: 项目开始日期

9. 结束日期 (End Date)
   - 类型: 日期 (Date)
   - 说明: 项目结束日期（进行中的项目可留空）

字段名称别名（脚本会自动匹配以下任一名称）：
------------------------------------------
- 项目名称: Title, 标题
- 项目描述: Description, 描述
- 状态: Status
- 技术栈: Technologies, Tech Stack
- 项目亮点: Highlights, 亮点
- GitHub: Github URL, 代码链接
- Demo: Demo URL, 演示链接
- 开始日期: Start Date
- 结束日期: End Date


=====================================================
三、AI 时间轴数据库 (AI Timeline Database)
=====================================================

数据库名称：AI 时间轴 / AI Timeline

必需字段：
-----------
1. 事件 (Title)
   - 类型: 标题 (Title)
   - 说明: AI 事件的标题

2. 时间 (Date)
   - 类型: 日期 (Date)
   - 说明: 事件发生的日期

推荐字段：
-----------
3. 描述 (Description)
   - 类型: 富文本 (Rich Text)
   - 说明: 事件的详细描述

4. 分类 (Category)
   - 类型: 选择 (Select)
   - 说明: 事件分类
   - 建议选项:
     * 架构创新
     * 预训练模型
     * 大模型时代
     * 应用突破
     * 多模态
     * 开源生态

5. 影响 (Impact)
   - 类型: 富文本 (Rich Text)
   - 说明: 该事件的影响和重要性

字段名称别名（脚本会自动匹配以下任一名称）：
------------------------------------------
- 事件: Title, 标题
- 时间: Date, 日期
- 描述: Description, 事件描述
- 分类: Category, 类别
- 影响: Impact, 重要性


=====================================================
四、使用说明
=====================================================

1. 在 Notion 中创建三个数据库（或使用现有数据库）
2. 按照上述字段配置创建相应的属性
3. 字段名称可以使用中文或英文，脚本会自动识别别名
4. 获取数据库 ID：
   - 打开 Notion 数据库页面
   - 复制浏览器地址栏中的 ID（问号前的32位字符）
   - 例如: notion.so/myworkspace/29aec685df0f801abe9efc95af9af44c?v=...
     数据库 ID 就是: 29aec685df0f801abe9efc95af9af44c

5. 配置环境变量或修改 scripts/fetch-notion-data.js：
   - NOTION_TOKEN: 你的 Notion Integration Token
   - NOTION_PAPERS_DATABASE_ID: 论文数据库 ID
   - NOTION_PROJECTS_DATABASE_ID: 项目数据库 ID
   - NOTION_AI_TIMELINE_DATABASE_ID: AI时间轴数据库 ID

6. 运行脚本获取数据：
   npm run fetch-data

7. 构建项目：
   npm run build


=====================================================
五、字段优先级说明
=====================================================

脚本会按照以下优先级尝试匹配字段：
1. 优先匹配中文字段名
2. 其次匹配英文字段名
3. 最后匹配别名

例如对于论文标题字段，会按顺序尝试：
标题 → Title → 论文标题

如果以上都不存在，该字段会使用默认值或留空。
